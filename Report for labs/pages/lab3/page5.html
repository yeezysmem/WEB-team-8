Висновки (спільні):

Способ вёрстки, основанный на таблицах, не неправильный, но не рекомендуется.
В общем случае таблицы следует использовать не для вёрстки страниц, а для отображения табличных данных на страницах.

Вёрстка, на div-ах со свойством float - наиболее используемый и рекомендуемый способ создания макетов страниц.
Альтернативно, для создания современных гибких макетов также используют flexbox (в CSS3).

На цій лабораторній роботі ми навчилися будувати макети HTML-сторінок за допомогою таблиць та плаваючих блоків. В ході виконання роботи побудували два макети HTML-сторінки цими двома способами. Висновки (Фоков Володимир):

Вёрстка, основанная на таблицах

Преимущества:

  • - её простота для верстальщика
  • - не нужно больших знаний CSS

Недостатки:

  • - большой минус - громоздкость итогового html-кода и связанная с этим плохая читаемость кода
  • - некоторые браузеры не показывают таблицу, пока не загружен тег её закрытия. Таким образом, пока не загрузится весь текст страницы, вы будете видеть только фоновый цвет

Вёрстка, основанная на тегах со свойством float

Преимущества:

  • - простота html-кода
  • - CSS-файл загружается один раз за всё время посещения сайта - для многих страниц. Следовательно, в целом скорость загрузки страниц будет более высокой.
  • - Страница показывается пользователю по мере загрузки
  • - Можно поменять макет для всех страниц сайта одновременно, отредактировав всего лишь один CSS-файл

Недостатки:

  • - необходимость достаточно глубоких знаний CSS
  • - для корректного отображения во всех браузерах выходит ещё больше css-кода

Висновки (Жданов Даніїл):

При вёрстке на блочных элементах легко добавить дополнительные внутренний блоки, в то время как в таблице пришлось бы добавлять дополнительный ряд/колонку и редактировать rowspan и colspan многих других элементов таблицы
